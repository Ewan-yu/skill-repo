# 图片位置处理指南

## 核心原则

**脚本输出就是真相**：`extract_content.js` 脚本返回的 `[IMG_N]` 占位符位置就是图片在原文中的正确位置。必须严格按照这个顺序和位置替换，不要自行添加或移动任何图片。

## 工作流程

### 1. 提取阶段

```bash
# 提取内容（包含 [IMG_N] 占位符）
agent-browser eval --stdin < scripts/extract_content.js

# 提取图片URL
agent-browser eval --stdin < scripts/extract_images.js
```

**关键点**：
- `extract_content.js` 返回的内容中，`[IMG_1]` 对应 `extract_images.js` 返回的第一张图片
- 图片顺序由 DOM 遍历顺序决定，与原文完全一致

### 2. 替换阶段

**正确做法**：
```markdown
# 脚本返回
正文内容...

[IMG_1]

更多内容...

[IMG_2]

结尾...

# 替换后
正文内容...

![数据表格](assets/01_wenzhangming/img01.png)

更多内容...

![走势图](assets/01_wenzhangming/img02.png)

结尾...
```

**错误做法**：
```markdown
# 错误1：在脚本输出之外添加图片
## 某个章节

![额外图片](assets/01_wenzhangming/img03.png)  # ❌ 不要这样做！

正文内容...

[IMG_1]

# 错误2：移动占位符位置
更多内容...

[IMG_1]  # ❌ 原本在"更多内容"前面，不要移动！

正文内容...

# 错误3：添加脚本未返回的图片
[IMG_1]

[IMG_2]

[IMG_3]  # ❌ 如果脚本只返回2张图片，不要添加第3张！

结尾...
```

### 3. 验证阶段

生成 Markdown 文件后，必须验证：

1. **数量验证**：图片数量 = `extract_images.js` 返回的数量
2. **顺序验证**：图片顺序与 `extract_images.js` 返回的顺序一致
3. **位置验证**：每张图片都在脚本指定的位置，没有额外添加的图片
4. **描述验证**：图片描述根据上下文生成，非通用描述

## 常见错误案例

### 案例1：在章节开头添加图片

**错误场景**：用户采集的文章中，某个章节看起来"需要"一张图片，于是在章节开头添加了一张。

**正确做法**：不要添加！脚本没有返回占位符，说明原文在这个位置没有图片。

### 案例2：移动图片到"更合适"的位置

**错误场景**：脚本返回的图片位置在段落中间，用户觉得放在段落开头"更合适"。

**正确做法**：不要移动！脚本返回的位置就是原文中的位置。

### 案例3：重复使用已下载的图片

**错误场景**：重新采集时，为了"节省空间"，将已下载的图片重复使用。

**正确做法**：重新采集时，应该按照脚本返回的图片URL重新下载，即使文件名相同。

## 重新采集时的特殊处理

当重新采集已存在的文章时：

1. **不要假设图片位置不变**：即使之前采集过，也要重新执行 `extract_content.js` 获取最新位置
2. **URL映射表只用于去重**：URL映射表只用于避免重复下载已存在的图片，不用于确定图片位置
3. **严格遵循脚本输出**：无论之前的文章文件如何，都要严格按照当前脚本输出生成新的 Markdown 文件

## 调试技巧

如果图片位置出现错误，可以通过以下方式调试：

1. **对比脚本输出与生成的文件**：
   - 保存 `extract_content.js` 的原始输出
   - 对比生成的 Markdown 文件
   - 找出差异

2. **检查图片数量**：
   ```bash
   # 统计脚本返回的图片数量
   echo "脚本返回的图片数量: $(grep -o '\[IMG_[0-9]*\]' script_output.txt | wc -l)"
   
   # 统计实际文件中的图片数量
   echo "文件中的图片数量: $(grep -o '!\[' article.md | wc -l)"
   ```

3. **检查图片顺序**：
   ```bash
   # 列出脚本返回的图片占位符
   grep -o '\[IMG_[0-9]*\]' script_output.txt
   
   # 列出文件中的图片
   grep -o '!\[.*\](assets/[^)]*)' article.md
   ```
