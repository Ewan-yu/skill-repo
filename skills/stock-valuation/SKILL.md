## 核心原则

### 数据真实原则
- **绝不编造数据**：所有数据必须来自权威数据源，严禁估算或猜测任何数值
- **数据优先级**：mx-data（结构化数据）> mx-search（资讯研报）> MCP工具（历史数据）> 询问用户
- **数据验证**：关键数据需交叉验证，确认准确性后再使用
- **明确标注**：数据来源和分析假设需清晰标注
- **禁止行为**：
  - ❌ 禁止编造或估算股价、PE、PB等任何数值
  - ❌ 禁止猜测大盘点位（必须通过mx-data或MCP查询）
  - ❌ 禁止使用"约""大概"等模糊词修饰具体数值
  - ❌ 禁止使用MCP历史估值数据作为当前股价（MCP的get_valuation_metrics可能返回过时数据）

### 估值严谨原则
- **方法适配**：根据行业特性和公司特点选择估值方法
- **历史对比**：必须拉取3-5年历史数据进行趋势分析
- **情景分析**：悲观/中性/乐观三情景估值
- **风险揭示**：充分揭示估值风险和不确定性

### 持续更新原则
- **结合最新信息**：利用mx-search获取最新研报、行业趋势
- **动态调整**：根据行业景气度变化灵活调整估值方法
- **证据支撑**：任何调整必须有最新数据和投研观点支持

## 核心投资框架：好生意、好公司、好价格

> 价值投资的三大支柱，缺一不可。详细内容参见：[references/investment-framework.md](references/investment-framework.md)

### 核心逻辑
```
好生意 → 好公司 → 好价格
  ↓        ↓        ↓
决定天花板  决定兑现  决定收益
```

### 三者关系
1. **好生意**是好公司的前提
2. **好公司**是好价格的基础
3. **三者兼备**才能长期获利

### 快速判断标准

| 维度 | 核心问题 | 关键指标 |
|------|---------|---------|
| **好生意** | 这是不是一个好行业？ | ROE>15%、强护城河、好竞争格局、高现金流 |
| **好公司** | 这是不是一家好企业？ | 优秀管理层、良好财务、核心竞争力、良好治理 |
| **好价格** | 现在是不是好价格？ | 估值合理、有安全边际、买入时机 |

## 行业研究框架速查

> 不同行业有不同研究重点和估值方法。详细内容参见：[references/industry-research.md](references/industry-research.md)

### 行业研究五层结构

1. **行业分类**：周期/非周期？消费/科技/金融/资源？
2. **宏观逻辑**：政策方向、利率、流动性影响
3. **产业逻辑**：行业空间、发展阶段、驱动变量
4. **价值链**：找最值钱的一环（竞争格局最好、壁垒最高）
5. **人和财务**：管理层、财务质量、盈利真实性

### 按行业选择估值方法

| 行业 | 主要方法 | 合理区间 | 关键指标 |
|------|----------|----------|----------|
| **白酒** | PE | 35-50倍 | PE、增速、品牌力 |
| **调味品** | PE | 50-70倍 | PE、渠道力 |
| **银行** | PB | 0.8-1.5倍 | PB、ROE、资产质量 |
| **保险** | PEV | 0.8-1.5倍 | PEV、新业务价值 |
| **医药-仿制药** | PE | 15-25倍 | PE、集采影响 |
| **医药-创新药** | PEG/rNPV | PEG<1.5 | PEG、管线价值 |
| **医药-CRO/CDMO** | PS | 8-15倍 | PS、增速、订单 |
| **科技-软件** | PS | 10-20倍 | PS、增速、续费率 |
| **科技-硬件** | PE | 20-40倍 | PE、增速、壁垒 |
| **周期股** | PB | 看位置 | PB、周期位置 |

### 按盈利情况选择

| 盈利情况 | 适用方法 | 示例 |
|----------|----------|------|
| 盈利稳定 | PE | 白酒、家电 |
| 盈利波动 | PB | 银行、周期 |
| 高成长 | PEG | 科技、医药 |
| 微利/亏损 | PS | 软件、互联网 |

## 估值方法体系

> 详细估值方法参见：[references/valuation-methods.md](references/valuation-methods.md)

### 核心公式速查

```
PE（市盈率）= 股价 ÷ 每股收益(EPS) = 总市值 ÷ 净利润
PB（市净率）= 股价 ÷ 每股净资产(BPS) = 总市值 ÷ 净资产
PS（市销率）= 总市值 ÷ 营业收入
PEG = PE ÷ 净利润复合增速
PEV = 总市值 ÷ 内含价值（保险专用）
```

### 估值计算公式

```
目标价 = 预测EPS × 目标PE
目标市值 = 预测净利润 × 目标PE

目标价 = 预测BPS × 目标PB
目标市值 = 预测净资产 × 目标PB

目标市值 = 预测收入 × 目标PS
```

### 银行特有公式

```
合理PB ≈ ROE × 1（理论值）
实际PB = ROE × 0.8~1.2（调整后）
```

### PEG判断标准

```
PEG < 1  → 低估
PEG = 1~1.5 → 合理
PEG > 1.5 → 高估
```

### 相对估值标准动作

1. 先确认企业适用哪种方法
2. 预测未来1-3年关键财务指标
3. 给目标估值倍数
4. 算出目标市值
5. 再算出目标价
6. 最后做悲观、中性、乐观三套情景

### 估值陷阱识别

| 陷阱类型 | 表象 | 真相 | 应对 |
|---------|------|------|------|
| **低PE陷阱** | PE 5-10倍 | 周期股盈利高点 | 周期股看PB不看PE |
| **伪成长陷阱** | 高PE高增速 | 增速不可持续 | 分析增长可持续性 |
| **价值陷阱** | PE很低 | 逻辑变坏 | 分析低估值原因 |
| **历史估值陷阱** | 历史底部 | 逻辑可能变坏 | 先检查逻辑是否变化 |

#### 2.1 必须先获取实时行情（mx-data）

**关键步骤**：在开始任何分析前，必须首先使用mx-data获取最新行情数据。

**执行顺序（不可颠倒）**：
1. `mx-data "[股票代码] 最新行情"` → 获取当前股价、涨跌幅、PE/PB
2. `mx-data "上证指数 最新点位"` → 获取大盘指数

**为什么mx-data是首选？**
- ✅ **实时性**：提供当日最新数据，避免使用过时信息
- ✅ **准确性**：结构化数据，来源权威可靠
- ✅ **完整性**：涵盖行情、财务、估值、股东等全方位信息

- **不要跳过此步骤直接使用MCP**：MCP的get_valuation_metrics可能返回过时数据

### 第三步：行业识别与估值方法选择

#### 估值方法决策树

```
盈利是否稳定？
├── 是 → 盈利增速如何？
│   ├── 稳定(0-15%) → PE估值
│   ├── 中速(15-30%) → PEG估值
│   └── 高速(>30%) → PEG或PS估值
└── 否 → 是否亏损？
    ├── 是 → PS或PB估值
    └── 否（波动大）→ PB估值

是否属于特殊行业？
├── 银行 → PB估值（ROE-PB关系）
├── 保险 → PEV估值
├── 创新药 → rNPV估值
└── 周期股 → PB估值（周期位置）
```

#### 结合最新研报调整估值方法

**搜索关键词**：
```
"[行业名称] 最新 估值方法 逻辑 变化"
"[股票代码] 估值框架 机构观点"
```

**关注调整信号**：
- 行业景气度变化
- 政策导向变化
- 技术变革影响
- 竞争格局变化
- 会计准则变化

### 第四步：历史数据分析

#### 必须分析的历史数据

**盈利能力分析**（至少3-5年）：
- 营业收入及增速
- 归母净利润及增速
- 毛利率、净利率变化
- EPS（每股收益）趋势

**财务数据获取要求**：
> ⚠️ **重要**：优先使用年度财务数据，而非季度推算数据。

**获取方式**：
1. **首选 mx-data**：`mx-data "[股票代码] 年度财务数据"` 或 `mx-data "[股票代码] 财务数据"`
2. **备选 MCP**：`get_income_statement` 获取利润表（至少8个季度，可推算年度）
3. **备选 MCP**：`get_balance_sheet` 获取资产负债表（至少8个季度）

**数据质量要求**：
- ✅ 优先使用完整年度数据（年报）
- ⚠️ 如仅有季度数据，需在报告中说明"基于季度数据推算"
- ❌ 禁止使用单季度数据代表全年

**财务健康度分析**：
- 资产负债率
- 流动比率、速动比率
- 经营现金流
- 应收账款占比

**估值指标历史**：
> ⚠️ **重要**：历史估值分位数需要足够的时间窗口，建议至少1年（250个交易日）。

**获取方式**：
1. **首选 mx-data**：`mx-data "[股票代码] 估值指标历史"`（如支持）
2. **备选 MCP**：`get_valuation_metrics` 获取历史估值指标

**时间窗口要求**：
- ✅ **推荐**：近1年（250个交易日）- 能反映完整市场周期
- ⚠️ **可接受**：近6个月（120个交易日）- 基本可用
- ❌ **不推荐**：近3个月（60个交易日）- 时间太短，分位数意义有限

**分位数计算说明**：
```
当前分位数 = (小于当前值的天数) / 总天数 × 100%

判断标准：
- 0-20%：历史低位（低估区域）
- 20-40%：偏低
- 40-60%：合理中枢
- 60-80%：偏高
- 80-100%：历史高位（高估区域）
```

**股东回报分析**：
- ROE（净资产收益率）
- 分红率、股息率

### 第五步：估值计算与情景分析

#### 三情景估值框架

**悲观情景**：
- 假设：行业下行、竞争加剧、增速放缓
- 盈利预测：保守（低于当前增速）
- 估值倍数：历史低位
- 得出：悲观目标价

**中性情景**：
- 假设：行业平稳、维持当前增速
- 盈利预测：中性（维持或略高于当前）
- 估值倍数：历史中枢
- 得出：中性目标价

**乐观情景**：
- 假设：行业上行、份额提升、增速加快
- 盈利预测：乐观（高于当前增速）
- 估值倍数：历史高位
- 得出：乐观目标价

**情景结果汇总**：
```
| 情景 | 目标价 | 相对当前 | 评价 |
|------|--------|----------|------|
| 悲观 | XX元 | -XX% | 安全边际 |
| 中性 | XX元 | -XX% | 合理估值 |
| 乐观 | XX元 | +XX% | 乐观预期 |
| 当前 | XX元 | - | - |
```

### 第六步：风险提示与投资建议

#### 必须提示的风险

**估值风险**：
- 当前估值相对历史位置
- 估值假设的合理性
- 情景实现的不确定性

**行业风险**：
- 行业周期位置
- 竞争格局变化
- 政策监管风险

**公司风险**：
- 财务质量风险
- 公司治理风险
- 关键人员风险

#### 投资建议框架

**不同投资者类型建议**：

| 投资者类型 | 建议内容 |
|------------|----------|
| 保守型 | 是否参与、安全边际要求 |
| 稳健型 | 仓位建议、入场时机 |
| 激进型 | 操作策略、风险控制 |

**操作建议**：
- 已持有者：继续/减仓/清仓
- 未持有者：观望/等待/参与
- 合理买入区间
- 止损/止盈位置

## 常见问题处理

### Q0: 为什么必须使用 mx-data 和 mx-search？

**核心原因**：
1. **实时性**：mx-data 提供当日最新数据，MCP 数据可能过时
2. **准确性**：mx-data 是结构化数据，来源权威可靠
3. **完整性**：mx-data 涵盖行情、财务、估值、股东等全方位信息
4. **时效性**：mx-search 提供最新研报和市场观点，对估值决策至关重要

**工具类型说明**：
- **mx-data 和 mx-search 是妙想Skills**（东方财富开发的独立技能），不是MCP服务
- 调用方式与MCP不同，请使用对应的技能调用方式

**工具使用优先级**：
```
mx-data 妙想Skill（实时行情、财务数据、估值指标）
    ↓
mx-search 妙想Skill（最新研报、行业趋势、公司动态）
    ↓
MCP（仅在上述工具不可用时作为备选）
```

**错误做法**：
- ❌ 跳过 mx-data 直接使用 MCP 获取当前股价
- ❌ 跳过 mx-search 直接使用 MCP 获取行业信息
- ❌ 在 mx 工具可用的情况下使用 MCP 作为主要数据源
- ❌ 将 mx-data 和 mx-search 误判为MCP服务而跳过使用

**正确做法**：
- ✅ 首先尝试使用 mx-data 妙想Skill 获取实时数据
- ✅ 然后使用 mx-search 妙想Skill 获取最新资讯
- ✅ 仅在工具不可用时，才使用 MCP 作为补充

### Q1.5: 如何判断 mx-data 和 mx-search 工具是否可用？

**重要说明**：mx-data 和 mx-search 是**妙想Skills**（东方财富开发），不是MCP服务。

**判断方法**：
1. **查看可用工具列表**：在可用工具列表中查找 mx-data 和 mx-search
2. **尝试调用 mx-data**：如果工具可用，会返回数据；如果不可用，会提示工具不存在
3. **尝试调用 mx-search**：同上

**工具可用时的行为**：
- mx-data：返回实时行情、财务数据、估值指标等结构化数据
- mx-search：返回最新研报、行业趋势、公司动态等资讯信息

**工具不可用时的行为**：
- 提示工具不存在或无法调用
- 此时才考虑使用 MCP 作为备选

**重要提醒**：
- 即使工具调用失败，也应在报告中说明"已尝试使用 mx-data/mx-search，但工具不可用，转用 MCP"
- 这样可以证明已按照技能要求优先尝试使用 mx 工具

### Q2: 如何获取大盘点位数据？

**正确做法**：
1. **首选mx-data**：`mx-data "上证指数 最新点位"`
2. **备选MCP**：`get_market_overview` 获取主要指数行情
3. **禁止**：❌ 禁止根据记忆猜测或估算大盘点位

### Q2: 股价数据出现偏差或过时怎么办？

**问题表现**：
- 不同数据源股价偏差 > 5%
- PE/PB数值异常（PE>200，PB>20）
- 数据时间戳 > 1天

**处理步骤**：
1. **立即重新获取**：使用mx-data获取最新行情
2. **交叉验证**：对比不同数据源的股价
3. **使用最新数据**：以时效性最好的数据源为准（mx-data > MCP）
4. **记录差异**：在报告中注明不同数据源的差异
5. **分析原因**：确认是数据延迟、数据源问题还是计算错误

**历史案例警示**：
- 中科曙光报告曾使用MCP数据导致股价偏差74%（53.32元 vs 实际92.76元）
- 原因：MCP的get_valuation_metrics返回了过时数据
- 教训：当前股价必须使用mx-data获取，MCP仅用于历史分位数

### Q3: 如何获取完整的年度财务数据？

**获取方式**：
1. **首选 mx-data**：`mx-data "[股票代码] 年度财务数据"` 或 `mx-data "[股票代码] 财务数据"`
2. **备选 MCP**：`get_income_statement` 获取利润表（至少8个季度，可推算年度）
3. **备选 MCP**：`get_balance_sheet` 获取资产负债表（至少8个季度）

**数据质量要求**：
- ✅ 优先使用完整年度数据（年报）
- ⚠️ 如仅有季度数据，需在报告中说明"基于季度数据推算"
- ❌ 禁止使用单季度数据代表全年

**报告标注要求**：
```
财务数据来源：
- 数据来源：cn-financial MCP
- 数据类型：2023-2025年年度财务数据（基于季度数据推算）
- 报告期：2025年12月31日
```

### Q4: 如何获取足够时间窗口的估值历史分位数？

**时间窗口要求**：
- ✅ **推荐**：近1年（250个交易日）- 能反映完整市场周期
- ⚠️ **可接受**：近6个月（120个交易日）- 基本可用
- ❌ **不推荐**：近3个月（60个交易日）- 时间太短，分位数意义有限

**获取方式**：
1. **首选 mx-data**：`mx-data "[股票代码] 估值指标历史"`（如支持）
2. **备选 MCP**：`get_valuation_metrics` 获取历史估值指标（默认100个交易日）

**分位数计算说明**：
```
当前分位数 = (小于当前值的天数) / 总天数 × 100%

判断标准：
- 0-20%：历史低位（低估区域）
- 20-40%：偏低
- 40-60%：合理中枢
- 60-80%：偏高
- 80-100%：历史高位（高估区域）
```

**报告标注要求**：
```
估值指标历史分位数：
- 数据来源：cn-financial MCP get_valuation_metrics
- 时间窗口：近100个交易日（2025-01-01 至 2026-05-05）
- 当前PE(TTM)：20.97倍
- 当前分位数：约65%（处于合理偏高位置）
```

### Q5: 搜索不到股票数据

**处理**：
1. 确认股票代码是否正确（6位数字）
2. 尝试用股票名称搜索
3. 检查是否为退市或新上市股票
4. 询问用户提供股票代码

### Q6: 数据不一致

**处理**：
1. 对比不同数据源
2. 查看数据时间戳
3. 优先使用最新官方数据
4. 在报告中注明差异

### Q5: 估值方法难以选择

**处理**：
1. 搜索最新研报看机构用什么方法
2. 用多种方法交叉验证
3. 在报告中说明选择理由
4. 展示不同方法的结果对比

### Q6: 数据显示异常高估/低估

**处理**：
1. 检查是否是盈利低谷期/高峰期
2. 查看是否有特殊事件影响
3. 分析是否是概念炒作
4. 在报告中充分揭示风险

### Q7: MCP的get_valuation_metrics能否用于获取当前股价？

**❌ 禁止使用**：MCP的get_valuation_metrics返回的股价数据可能过时

**正确用法**：
- **仅用于历史分位数分析**：PE/PB的历史最小值、25%分位、50%分位等
- **当前股价必须用mx-data**：`mx-data "[股票代码] 最新行情"`
- **如需验证**：可用get_realtime_quote交叉验证，但仍以mx-data为准
