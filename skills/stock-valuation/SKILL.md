# 股票估值分析技能

> 基于价值投资入门与实践方法论，结合妙想Skills（mx-data+mx-search）和MCP数据源的A股估值分析系统

---

## 核心原则

### 数据真实原则
- **绝不编造数据**：所有数据必须来自权威数据源，严禁估算或猜测任何数值
- **数据优先级**：mx-data（结构化数据）> mx-search（资讯研报）> MCP工具（历史数据）> 询问用户
- **数据验证**：关键数据需交叉验证，确认准确性后再使用
- **明确标注**：数据来源和分析假设需清晰标注
- **禁止行为**：
  - ❌ 禁止编造或估算股价、PE、PB等任何数值
  - ❌ 禁止猜测大盘点位（必须通过mx-data或MCP查询）
  - ❌ 禁止使用"约""大概"等模糊词修饰具体数值
  - ❌ 禁止使用MCP历史估值数据作为当前股价（MCP的get_valuation_metrics可能返回过时数据）

### ⚠️ 数据验证机制（防止数据错误）

**股价数据验证流程**（必须执行）：
1. **第一步**：使用mx-data获取最新行情：`mx-data "[股票代码] 最新行情"`
2. **第二步**：记录当前日期和股价，确认数据时效性
3. **第三步**：如需MCP补充历史数据，仅使用get_realtime_quote和get_valuation_metrics的历史分位数
4. **第四步**：交叉验证 - 对比mx-data和MCP的当前股价，偏差>5%时需调查原因
5. **第五步**：在报告中明确标注数据来源和获取时间

**数据异常识别标准**：
- 股价偏差 > 5%：需调查数据源差异
- PE/PB极端值（PE>200或<0，PB>20）：需验证数据准确性
- 不同数据源冲突：以最新权威数据源为准
- 数据时间戳 > 1天：需重新获取最新数据

**数据时效性要求**：
- 股价、涨跌幅：必须使用当日数据
- 估值指标（PE/PB）：必须使用最新交易日数据
- 财务数据：注明报告期（如2025年Q3）
- 大盘指数：当日最新点位

### 妙想Skills使用原则

**数据获取策略**：妙想Skills提供两大核心能力，优先使用妙想获取数据，不足时再使用MCP工具补充。

| 工具 | 核心功能 | 使用场景 |
|------|---------|---------|
| **mx-data** | 获取股票、板块、指数、基金、债券的行情、主力资金、估值等数据；公司基本信息、财务、高管、主营业务、股东、融资等数据 | ①获取公司基本信息和主营业务<br>②获取实时行情和历史股价<br>③获取财务数据（三张报表）<br>④获取估值指标（PE/PB/PS等）<br>⑤获取股东和高管信息 |
| **mx-search** | 获取实时新闻、事件跟踪、舆情监控、行业动态等 | ①搜索最新研报和机构观点<br>②搜索行业趋势和景气度<br>③搜索公司重大事件和公告<br>④搜索管理层访谈和观点<br>⑤搜索政策法规影响 |

### 估值严谨原则
- **方法适配**：根据行业特性和公司特点选择估值方法
- **历史对比**：必须拉取3-5年历史数据进行趋势分析
- **情景分析**：悲观/中性/乐观三情景估值
- **风险揭示**：充分揭示估值风险和不确定性

### 持续更新原则
- **结合最新信息**：利用mx-search获取最新研报、行业趋势
- **动态调整**：根据行业景气度变化灵活调整估值方法
- **证据支撑**：任何调整必须有最新数据和投研观点支持

## 核心投资框架：好生意、好公司、好价格

> 价值投资的三大支柱，缺一不可

### 一、好生意：商业模式分析

**好生意的定义**：能够持续产生高额自由现金流的商业模式

#### 1.1 好生意的五大特征

| 特征 | 评估标准 | 为什么重要 |
|------|----------|-----------|
| **高ROE** | 持续ROE > 15% | 反映资本回报能力 |
| **强护城河** | 品牌/网络效应/成本优势/转换成本 | 防止竞争侵蚀利润 |
| **好竞争格局** | 寡头或一超多强，CR3 > 50% | 避免恶性价格战 |
| **高现金流** | 经营现金流/净利润 > 80% | 真金白银而非会计利润 |
| **持续成长** | 行业扩容或份额提升 | 长期复利基础 |

#### 1.2 护城河类型识别

| 护城河类型 | 检验方法 | 代表行业 |
|-----------|---------|---------|
| **品牌护城河** | 能否提价不丢失客户？ | 高端白酒、奢侈品 |
| **网络效应** | 用户越多价值越大？ | 社交平台、支付 |
| **转换成本** | 客户难以切换供应商？ | 企业软件、银行系统 |
| **成本优势** | 规模或成本领先？ | 零售、制造业 |
| **监管牌照** | 政府稀缺牌照？ | 银行、保险、免税 |

#### 1.3 按商业模式分类研究

| 商业模式类型 | 特征 | 研究重点 | 代表行业 |
|-------------|------|---------|---------|
| **消费型** | TO C，品牌驱动 | 品牌、渠道、产品力 | 白酒、家电、食品 |
| **生产型** | TO B，成本驱动 | 成本、产能、技术 | 化工、钢铁、有色 |
| **服务型** | TO C/B，服务驱动 | 流量、转化、留存 | 银行、保险、软件 |
| **平台型** | 网络效应 | 用户规模、网络效应 | 互联网、平台 |
| **资源型** | 资源驱动 | 资源储量、价格 | 矿产、能源 |

#### 1.4 按成长属性分类估值

| 成长类型 | 增速特征 | 估值方法 | 研究重点 |
|----------|----------|---------|---------|
| **高成长** | > 30% | PS、PEG | 成长空间、护城河 |
| **稳定成长** | 10-30% | PE、PEG | 竞争优势、持续性 |
| **低成长** | < 10% | PE、PB、股息率 | 价值、安全边际 |
| **周期型** | 波动大 | PB、周期位置 | 周期位置、供需 |

#### 1.5 按生命周期分类研究

| 生命周期 | 特征 | 估值方法 | 研究重点 |
|-----------|------|---------|---------|
| **导入期** | 市场小、增速快、风险大 | PS、期权价值 | 技术可行性、市场接受度 |
| **成长期** | 市场扩容、竞争加剧 | PEG、PS、DCF | 竞争格局、护城河 |
| **成熟期** | 增速放缓、格局稳定 | PE、PB、DCF | 市场份额、盈利能力 |
| **衰退期** | 需求下滑 | PB、清算价值 | 转型、现金流 |

### 二、好公司：企业分析框架

**好公司定义**：在好生意的基础上，拥有优秀管理层、良好财务、强大执行力

#### 2.1 管理层评估

| 评估维度 | 关键问题 | 考察方法 |
|---------|---------|---------|
| **战略能力** | 战略是否清晰？是否符合趋势？ | 读历年年报、战略陈述 |
| **执行力** | 战略能否落地？业绩承诺能否兑现？ | 对比目标与实际完成情况 |
| **诚信度** | 财务是否真实？信息披露是否充分？ | 查询处罚记录、关联交易 |
| **激励机制** | 管理层和股东利益是否一致？ | 查看股权激励计划 |

**mx-search关键词**：
```
"[公司名] 管理层 访谈 战略"
"[公司名] 股权激励 激励机制"
"[公司名] 处罚 罚信 关联交易"
```

#### 2.2 财务健康评估

| 财务指标 | 好公司标准 | 差公司信号 | 获取方式 |
|---------|-----------|-----------|---------|
| **ROE** | > 15% | < 10% | MCP财务数据 |
| **资产负债率** | < 60% | > 80% | MCP资产负债表 |
| **流动比率** | > 1.5 | < 1 | MCP资产负债表 |
| **经营现金流** | > 净利润 | < 净利润 | MCP现金流量表 |
| **应收账款增速** | < 营收增速 | > 营收增速 | MCP利润表对比 |

#### 2.3 竞争力评估

| 竞争力类型 | 评估要点 | 数据来源 |
|-----------|---------|---------|
| **品牌** | 知名度、美誉度、定价权 | mx-search品牌排名 |
| **技术** | 专利数量、研发投入、技术壁垒 | mx-data/mx-search研发投入 |
| **渠道** | 覆盖率、控制力、深度 | mx-data渠道数据 |
| **成本** | 规模效应、成本结构 | MCP财务数据分析 |
| **网络效应** | 用户规模、粘性 | mx-data用户数据 |

#### 2.4 治理结构评估

| 治理维度 | 好公司 | 差公司 |
|---------|-------|-------|
| **股权结构** | 适度集中 | 一股独大或过度分散 |
| **董事会** | 独立董事比例高 | 独立董事是花瓶 |
| **信息披露** | 充分透明 | 遮遮掩掩 |
| **关联交易** | 少 | 频繁 |

### 三、好价格：估值与时机

**好价格定义**：估值合理或低估，有足够安全边际

#### 3.1 估值安全边际要求

| 公司类型 | PE标准 | 安全边际要求 |
|---------|--------|------------|
| **高成长稳定** | < 30倍 | 20-30% |
| **中等成长** | < 20倍 | 30-40% |
| **低成长/价值** | < 15倍 | 40%+ |

#### 3.2 买入时机识别

**✅ 好时机（符合以下任一）**：
- 市场恐慌时（大盘暴跌、情绪冰点）
- 好公司遭遇短期利空（逻辑未变）
- 行业阶段性低迷
- 错杀（基本面无变化）

**❌ 坏时机（符合以下任一）**：
- 估值泡沫化（远超历史区间）
- 市场狂热时（人人谈论股票）
- 追涨热点（短期炒作）
- 逻辑已变（基本面恶化）

#### 3.3 能力圈原则

> "知道自己知道什么，不知道什么" —— 巴菲特

**能力圈建立步骤**：
1. 选择1-2个行业深入研究
2. 研究行业内主要公司
3. 跟踪行业动态和公司变化
4. 形成自己的判断体系

**能力圈检验**：
- 能否清晰解释行业盈利模式？
- 能否判断行业未来3年趋势？
- 能否识别行业风险？
- 如有疑问，即能力圈外，应避免

## 行业研究框架速查

> 不同行业有不同研究重点和估值方法

### 消费行业研究框架

**核心逻辑**：品牌 + 渠道 + 产品

**研究框架**：
```
1. 品牌分析
   ├── 品牌力（知名度、美誉度、忠诚度）
   ├── 品牌定位（目标客群、价格带、差异化）
   └── 品牌趋势（提价能力、品牌年轻化）

2. 渠道分析
   ├── 渠道覆盖（铺市率、深度、广度）
   ├── 渠道控制（直销vs经销、渠道利润）
   └── 渠道趋势（线上化、渠道下沉）

3. 产品分析
   ├── 产品结构（核心、成长、培育产品）
   ├── 产品力（品质、包装、创新）
   └── 产品趋势（高端化、健康化）

4. 财务分析
   ├── ROE > 15%
   ├── 经营现金流/净利润 > 80%
   └── 存货周转率
```

**估值方法**：PE为主，高端白酒可给高PE（35-50倍）

### 医药行业研究框架

**核心逻辑**：创新 + 管线 + 商业化

**研究框架**：
```
1. 行业分析
   ├── 细分领域（创新药、仿制药、CRO、医疗器械）
   ├── 政策环境（医保、集采、审批）
   └── 行业趋势（创新导向、集采常态化）

2. 产品分析
   ├── 管线评估（数量、阶段分布、上市预期）
   ├── 核心产品（市场空间、竞争格局、专利保护）
   └── 研发能力（投入、团队、效率）

3. 商业化分析
   ├── 销售能力（团队、覆盖、学术推广）
   └── 市场准入（医保目录、基药目录、招标）

4. 估值分析
   ├── 创新药：rNPV、DCF
   ├── 仿制药：PE（15-25倍）
   └── CRO/CDMO：PS（8-15倍）
```

### 银行行业研究框架

**核心逻辑**：资产质量 + 息差 + 规模

**研究框架**：
```
1. 资产质量
   ├── 不良率（趋势、行业分布）
   ├── 拨备覆盖率（安全垫厚度）
   └── 关注类贷款占比

2. 盈利能力
   ├── 净息差（NIM）
   ├── 非息收入占比
   └── 成本收入比

3. 成长性
   ├── 规模增长
   ├── 息差趋势
   └── 中收增长

4. 估值分析
   └── PB估值：合理PB ≈ ROE × 1（调整0.8-1.2）
```

**估值方法**：PB为主（0.8-1.5倍），关注ROE-PB关系

### TMT行业研究框架

**核心逻辑**：技术 + 趋势 + 格局

**研究框架**：
```
1. 技术分析
   ├── 技术趋势（路线、周期、壁垒）
   ├── 研发能力（投入、专利、团队）
   └── 技术突破（新技术、新应用）

2. 产业分析
   ├── 产业周期（导入、成长、成熟）
   ├── 产业链（上游、中游、下游）
   └── 供需分析

3. 竞争分析
   ├── 竞争格局（玩家、份额、态势）
   ├── 竞争要素（技术、成本、规模）
   └── 进入壁垒（技术、规模、粘性）

4. 估值分析
   ├── 成长期：PS（10-20倍）、PEG
   └── 成熟期：PE（20-40倍）
```

### 周期行业研究框架

**核心逻辑**：周期位置 + 供需 + 价格

**研究框架**：
```
1. 周期判断
   ├── 当前位置（复苏、过热、衰退、谷底）
   ├── 驱动因素（供给、需求、库存）
   └── 周期时长（本轮持续多久？）

2. 供需分析
   ├── 产能投放（新增产能、投放节奏）
   ├── 需求变化（下游需求、替代品）
   └── 库存水平（厂商库存、社会库存）

3. 价格分析
   ├── 当前价格 vs 成本曲线
   ├── 历史价格分位数
   └── 盈利能力测算

4. 估值分析
   └── PB估值（看位置），PE在谷底/峰值失真
```

**估值方法**：PB为主，PE在极端位置时失真

## 工作流程

### 第一步：识别分析目标

明确用户需求：
- 个股代码/名称
- 分析目的（投资决策/学习/研究）
- 关注重点（估值/行业/财务）

### 第二步：数据获取（按优先级）

#### 2.1 【第一步】⭐ 必须先获取实时行情（mx-data）

**⚠️⚠️⚠️ 关键步骤**：在开始任何分析前，必须首先使用mx-data获取最新行情数据。

**执行顺序（不可颠倒）**：
1. `mx-data "[股票代码] 最新行情"` → 获取当前股价、涨跌幅、PE/PB
2. `mx-data "上证指数 最新点位"` → 获取大盘指数
3. 记录获取时间，确认数据时效性

**⚠️ 警告**：
- **不要跳过此步骤直接使用MCP**：MCP的get_valuation_metrics可能返回过时数据
- **历史案例**：中科曙光报告曾使用MCP数据导致股价偏差74%（53.32元 vs 实际92.76元）
- **验证原则**：任何来源的股价数据必须与mx-data最新行情对比验证

**必须使用mx-data获取的数据**：

| 数据类型 | mx-data搜索关键词示例 | 何时使用 |
|---------|---------------------|----------|
| **实时行情** | "[股票代码] 最新行情" 或 "[股票代码] 股价" | ⭐⭐⭐ 必须第一步执行，获取当前股价、涨跌幅 |
| **大盘指数** | "上证指数 最新点位" 或 "大盘 行情" | ⭐⭐⭐ 第二步执行，禁止猜测大盘点位 |
| **财务数据** | "[股票代码] 财务数据" 或 "[股票代码] 业绩" | ⭐ 首选，获取营收、利润、ROE等 |
| **估值指标** | "[股票代码] 估值指标 PE PB" | ⭐ 首选，获取最新PE/PB/PS |
| **公司信息** | "[股票代码] 公司资料" 或 "[股票代码] 基本信息" | 了解公司业务、行业、产品 |
| **股东高管** | "[股票代码] 股东结构" 或 "[股票代码] 高管" | 获取股东结构、高管团队 |
| **资金流向** | "[股票代码] 资金流向" 或 "[股票代码] 主力资金" | 获取主力资金流向 |
| **板块数据** | "[行业名称] 板块行情" 或 "[行业名称] 龙头" | 获取行业整体表现 |

**执行方式**：
```bash
# 使用mx-data技能（Skill工具调用）
# 示例：
mx-data "600519 最新行情"      # 第一步：获取当前股价
mx-data "上证指数 最新点位"      # 第二步：获取大盘指数

#### 3.2 估值方法决策树

```

├── 银行 → PB估值（ROE-PB关系）
├── 保险 → PEV估值
├── 创新药 → rNPV估值
└── 周期股 → PB估值（周期位置）

- EPS × 股本 ≈ 净利润

### 第五步：估值计算与情景分析

#### 5.1 主要估值方法

**PE估值**：
```
当前EPS = 归母净利润 ÷ 总股本
正常化EPS = 过去3年平均EPS（如有波动）
合理PE = 参考历史区间和行业地位
目标价 = 正常化EPS × 合理PE
```

**PB估值**：
```
每股BPS = 归母净资产 ÷ 总股本
合理PB = 参考历史区间和ROE水平
ROE-PB关系：合理PB ≈ ROE × 1（调整系数0.8-1.2）
目标价 = 每股BPS × 合理PB
```

**PS估值**：
```
每股营收 = 营业收入 ÷ 总股本
合理PS = 参考历史区间和行业特性
目标市值 = 营业收入 × 合理PS
```

**PEG估值**：
```
PEG = PE ÷ 预期增速
PEG < 1：低估
PEG = 1-1.5：合理
PEG > 1.5：高估
合理PE = 预期增速 × 目标PEG（1-1.5）
```

#### 5.2 三情景估值框架

**悲观情景**：
- 假设：行业下行、竞争加剧、增速放缓
- 盈利预测：保守（低于当前增速）
- 估值倍数：历史低位
- 得出：悲观目标价

**中性情景**：
- 假设：行业平稳、维持当前增速
- 盈利预测：中性（维持或略高于当前）
- 估值倍数：历史中枢
- 得出：中性目标价

**乐观情景**：
- 假设：行业上行、份额提升、增速加快
- 盈利预测：乐观（高于当前增速）
- 估值倍数：历史高位
- 得出：乐观目标价

**情景结果汇总**：
```
| 情景 | 目标价 | 相对当前 | 评价 |
|------|--------|----------|------|
| 悲观 | XX元 | -XX% | 安全边际 |
| 中性 | XX元 | -XX% | 合理估值 |
| 乐观 | XX元 | +XX% | 乐观预期 |
| 当前 | XX元 | - | - |
```

## 常见问题处理

### Q1: 如何获取大盘点位数据？

**正确做法**：
1. **首选mx-data**：`mx-data "上证指数 最新点位"`
2. **备选MCP**：`get_market_overview` 获取主要指数行情
3. **禁止**：❌ 禁止根据记忆猜测或估算大盘点位

### Q2: 股价数据出现偏差或过时怎么办？

**⚠️ 严重问题 - 必须立即处理**

**问题表现**：
- 不同数据源股价偏差 > 5%
- PE/PB数值异常（PE>200，PB>20）
- 数据时间戳 > 1天

**处理步骤**：
1. **立即重新获取**：使用mx-data获取最新行情
2. **交叉验证**：对比不同数据源的股价
3. **使用最新数据**：以时效性最好的数据源为准（mx-data > MCP）
4. **记录差异**：在报告中注明不同数据源的差异
5. **分析原因**：确认是数据延迟、数据源问题还是计算错误

**历史案例警示**：
- 中科曙光报告曾使用MCP数据导致股价偏差74%（53.32元 vs 实际92.76元）
- 原因：MCP的get_valuation_metrics返回了过时数据
- 教训：当前股价必须使用mx-data获取，MCP仅用于历史分位数

### Q3: 搜索不到股票数据

**处理**：
1. 确认股票代码是否正确（6位数字）
2. 尝试用股票名称搜索
3. 检查是否为退市或新上市股票
4. 询问用户提供股票代码

### Q4: 数据不一致

**处理**：
1. 对比不同数据源
2. 查看数据时间戳
3. 优先使用最新官方数据
4. 在报告中注明差异

### Q5: 估值方法难以选择

**处理**：
1. 搜索最新研报看机构用什么方法
2. 用多种方法交叉验证
3. 在报告中说明选择理由
4. 展示不同方法的结果对比

### Q6: 数据显示异常高估/低估

**处理**：
1. 检查是否是盈利低谷期/高峰期
2. 查看是否有特殊事件影响
3. 分析是否是概念炒作
4. 在报告中充分揭示风险

### Q7: MCP的get_valuation_metrics能否用于获取当前股价？

**❌ 禁止使用**：MCP的get_valuation_metrics返回的股价数据可能过时

**正确用法**：
- **仅用于历史分位数分析**：PE/PB的历史最小值、25%分位、50%分位等
- **当前股价必须用mx-data**：`mx-data "[股票代码] 最新行情"`
- **如需验证**：可用get_realtime_quote交叉验证，但仍以mx-data为准
