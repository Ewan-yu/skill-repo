## 报告输出规范

### 输出路径

```
analysis/valuation/[股票名称]([股票代码])_估值分析报告_[YYYYMMDD].md
```

**示例**：
```
analysis/valuation/双环传动(002472)_估值分析报告_20260505.md
```

### 文件命名规则

- 格式：`[股票名称]([股票代码])_估值分析报告_[YYYYMMDD].md`
- 日期使用报告生成日期

### 输出流程

1. 完成所有数据分析和估值计算
2. 按照报告模板生成完整报告内容
3. 使用Write工具将报告保存到本地文件
4. 向用户确认报告已保存，并告知文件路径

## 质量控制检查清单

在输出报告前，**必须**确认：

### 数据质量检查（最重要）

- [ ] 已使用mx-data获取最新行情数据
- [ ] 已记录数据获取时间
- [ ] 已验证股价数据时效性（当日或最近交易日）
- [ ] 如使用MCP补充数据，已与mx-data验证一致性
- [ ] 股价偏差<5%，否则需调查原因
- [ ] PE/PB数据在合理范围内（PE<200，PB<20）
- [ ] 数据来源已清晰标注

### 分析质量检查

- [ ] 估值方法选择有理有据
- [ ] 历史数据至少3年（或8个季度）
- [ ] 三情景估值完整（悲观/中性/乐观）
- [ ] 风险提示充分

### 输出质量检查

- [ ] 格式规范完整
- [ ] 计算过程清晰
- [ ] 结论明确不模糊
- [ ] 建议具体可操作
- [ ] 报告已保存到相对路径
