## ⚠️ 行为准则（必须严格遵守）

**这些规则优先级最高，违反任何一条都会导致用户得到错误信息：**

1. **始终使用 --minute 参数**。查询任何股票时，必须运行 `--minute` 获取分时量能分析。不要运行不带 `--minute` 的简化版。
2. **脚本输出是数据源，不是最终报告**。`analyze.py` 输出的章节是原始数据，Claude 必须基于这些数据生成 v5.0 融合报告（7个章节）。**禁止**原样展示脚本输出，**禁止**对数据进行摘要、省略或改写。
3. **不要降级输出**。无论上下文窗口多大，始终输出完整分析结果。不存在"摘要版"或"精简版"。
4. **单只股票默认行为**（含竞价量获取，共3步）：
   - Step 1: `python C:/Users/kense/.claude/skills/mx-data/mx_data.py "<股票名> 集合竞价数据"` 获取今日和前日竞价量
   - Step 2: 从返回的 `rawTable["100000000047336"]` 中提取今日（index 0）和前日（index 1）的竞价量（单位：股）
   - Step 3: `uv run {baseDir}/scripts/analyze.py <代码> --minute --auction-today-shares <今日股数> --auction-prev-shares <前日股数>`
5. **多只股票同时分析**: 对每只股票分别执行 Step 1-2，然后合并到一条 analyze.py 命令（每只股票的竞价量参数只对单股有效，多股时分别单独运行）。
6. **近期走势数据获取**。运行 analyze.py 后，使用 `mcp__cn-financial__get_historical_price` 获取最近20个交易日日K线数据（adjust="qfq"），用于生成近期走势章节和趋势评分。
7. **同时段对比必须包含量比和额比**。两个维度缺一不可。
8. **盘后复盘模式**：当用户明确要求复盘、或数据状态为"已收盘"时，主动提示用户是否需要 `--review` 参数。使用 `--review` 时脚本自动包含 `--minute`，无需重复传入。复盘模式下必须生成第七章"次日操作计划"。
9. **数据真实性原则（最高优先级）**：
   - 脚本输出中标注"无前日数据"、"无竞价数据"、"数据不足"的字段，**禁止**在报告中编造或推测对应的值。
   - 综合评分的 `missing_data` 列表不为空时，必须在报告中明确列出缺失项，并说明评分满分已相应降低。
   - 竞价量对比章节若显示"数据不足"，不得在报告中给出竞价强弱判断。
   - 预期差章节若显示"no_prior_signal"，不得推断预期差方向。
   - 历史K线获取失败时，趋势分标记为"无法计算"，总分不得用估算值填充。
   - **宁可告知用户数据不足、无法给出判断，也不能用猜测的数据支撑操作建议。**
10. **近期走势章节不绘制走势图**。只输出关键价位和走势定性文字描述，不画ASCII/K线图。

**为什么这些规则很重要**：用户在实际交易中依赖这些数据做买卖决策。v5.0 融合报告将原始数据转化为可执行的判断，如果省略关键章节（比如"竞价量对比"或"次日操作计划"），用户可能错过关键的交易信号，造成实际资金损失。

### mx-data 竞价量解析说明

mx-data 返回的 JSON 中，`rawTable` 字段结构如下：

```json
{
  "rawTable": {
    "100000000047336": ["719500", "564200", ...],  // 开盘集合竞价成交量（股）
    "100000000047337": ["5072万元", ...],           // 开盘集合竞价成交额
    "100000000041102": ["75.27万股", ...],          // 收盘集合竞价成交量
    "headName": ["2026-05-18", "2026-05-15", ...]  // 日期（index 0 = 最新）
  }
}
```

提取规则：`rawTable["100000000047336"][0]` = 今日竞价量（股），`[1]` = 前日竞价量（股）。
直接将字符串转为整数传入 `--auction-today-shares` 和 `--auction-prev-shares`。

## 盘中看盘场景

盘中分析的核心决策链（按优先级）：

1. **竞价量比** → 判断今日开盘强弱基础（脚本"竞价量对比"章节）
2. **预期差信号** → 判断是否存在博弈机会（脚本"预期差信号"章节）
3. **早盘30分占比+额比** → 确认主力参与度（脚本"早盘量能预期"章节）
4. **VWAP关系** → 实时买卖时机（白线在黄线上方=多头占优）
5. **综合评分** → 最终持仓/离场决策（脚本"综合评分"章节 + Claude补充趋势分）

**关键阈值（来自实战资料）**：

| 指标 | 强势信号 | 弱势信号 |
|------|---------|---------|
| 早盘占比 | ≥30% | <20%（今日基本无戏） |
| 竞价量比 | >1.3x | <0.7x（不好做T） |
| 价格 vs VWAP | 白线在黄线上方 | 跌破黄线且无法收复 |
| 综合评分 | ≥60/100 | <40/100（不介入） |

**盘中买入三原则**（来自实战资料）：
1. 必须是近期热点，不对没热过的板块做预判
2. 低吸不追高，介入不超过昨收+3%，最好绿盘介入
3. 超短就是超短，次日不强势涨停必须出局

## 盘后复盘场景

**触发条件**：用户明确要求复盘，或数据状态为"已收盘"。

**命令**：`uv run {baseDir}/scripts/analyze.py <代码> --review`

**复盘分析流程**：
1. 运行 `--review` 获取脚本复盘数据章节
2. 使用 `mcp__cn-financial__get_historical_price` 获取20日历史K线，计算趋势分
3. 汇总综合评分（100分制 = 脚本75分 + 趋势25分）
4. 基于 `references/short-term-trading-tactics.md` 生成次日操作计划

**趋势分计算规则**（Claude 基于历史K线计算）：
- 价格在60日线上且5日线向上 → 25分
- 价格在5日线上但低于60日线 → 15分
- 价格在5日线附近（±1%）→ 10分
- 价格在5日线下 → 5分

**盘后标的筛选标准**（次日关注股）：
- 综合评分 ≥ 60/100
- 今日涨停或涨幅 > 5%
- 5日线向上
- 属于近期热点板块

### 第四章关键信号的子模块

**竞价量对比**（来自脚本"竞价量对比"章节）：
- 展示今日 vs 前日竞价量、竞价量比
- 竞价量比 < 0.7 → 标注"缩量开盘，不好做T，冲高概率低"
- 竞价量比 > 1.3 → 标注"放量开盘，承接力强，冲高概率高"
- 无竞价数据时说明原因（非交易日或数据不足）

**预期差信号**（来自脚本"预期差信号"章节）：
- 仅在 prev_limit_up=True（前日涨停）时有实质内容
- below_expectation/strong → ⚠️ 强信号，附三部曲操作提示
- below_expectation/moderate → ⚡ 中信号
- above_expectation → 提示高开不追，等回踩
- no_prior_signal → 简短说明无预期差基础

**承接力多维验证**（来自脚本"承接力判断"章节）：
- 量价信号 + VWAP关系 + 额比三维度验证
- 每个维度标注 ✓/✗

**卖出三部曲信号**（来自脚本"卖出三部曲信号"章节）：
- ⚠️ 强信号：前日涨停+30分钟不红盘；冲高回落破VWAP
- ⚡ 中信号：前日未涨停+1小时不红盘；尾盘未涨停

### 第五章综合研判的评分展示

```
综合评分: XX/100
  量能: XX/25  承接: XX/25  预期差: XX/25  趋势: XX/25
评级: 强势(≥80) / 偏多(60-79) / 中性(40-59) / 偏空(<40)

```

## 分时量能分析说明

### 成交分布时段

| 时段 | 说明 |
|------|------|
| 早盘30分 (9:30-10:00) | 主力早盘动作，>30%为抢筹信号，<20%全天基本无戏 |
| 上午中段 (10:00-11:30) | 正常交易 |
| 下午中段 (13:00-14:30) | 正常交易 |
| 尾盘30分 (14:30-15:00) | >25%为尾盘异动信号 |

### 竞价量对比（v5.0 新增）

集合竞价量来自新浪分时数据 09:25 那根K线。

| 竞价量比 | 信号 | 含义 |
|---------|------|------|
| >1.3 | 放量开盘（强） | 承接力强，冲高概率高 |
| 1.0-1.3 | 温和放量（中） | 有一定承接 |
| 0.7-1.0 | 平量开盘（中） | 强弱待观察 |
| <0.7 | 缩量开盘（弱） | 不好做T，冲高概率低 |

### 预期差检测（v5.0 新增）

仅在前日涨停时有实质意义。

| 条件 | 预期差类型 | 操作参考 |
|------|-----------|---------|
| 前日涨停+今日低开>2% | below/strong | 30分钟不翻红立即卖出 |
| 前日涨停+今日低开+竞价缩量 | below/moderate | 密切关注30分钟内能否翻红 |
| 前日涨停+今日小幅低开 | below/weak | 等翻红后量能确认再介入 |
| 前日涨停+今日小幅高开 | meet | 正常观察 |
| 前日涨停+今日高开>3% | above | 高开不追，等回踩 |

### 同时段对比（v5.0 量比+额比双维度）

| 额比范围 | 趋势判断 | 含义 |
|----------|----------|------|
| >1.3 | 明显增量 | 资金积极参与 |
| 1.1-1.3 | 温和增量 | 量能温和放大 |
| 0.9-1.1 | 基本持平 | 量能稳定 |
| 0.7-0.9 | 温和缩量 | 资金观望 |
| <0.7 | 明显缩量 | 资金离场或惜售 |

额比比量比更重要：价格上涨时成交额同步放大，额比能更真实反映资金参与度。

### 承接力判断（v5.0 多维验证）

基于昨收价（而非当日第一根K线）判断量价配合：

| 信号 | 条件 | 判断 |
|------|------|------|
| 缩量回调 | 跌>0.5% + 额比<0.8 | 承接力强（主力护盘，散户惜售） |
| 放量下跌 | 跌>0.5% + 额比>1.2 | 承接力弱（主力出货，恐慌抛售） |
| 放量上涨 | 涨>0.5% + 额比>1.2 | 量价配合好（主力积极做多） |
| 缩量上涨 | 涨>0.5% + 额比<0.8 | 量价背离（主力控盘或散户推动） |

### VWAP（均价线）增强分析

成交量加权均价（分时图黄线），是实战中最重要的参考线。

**黄线走势方向判断**（v5.1新增）：
- **黄线向上**：均价成本持续上移，买方主动，积极做多
- **黄线横盘**：均价成本稳定，多空平衡，等待方向选择
- **黄线向下**：均价成本下移，卖方主动，谨慎观望
- **黄线震荡**：均价成本波动较大，多空激烈博弈

**白黄线关系判断**（v5.1新增）：
- **长时间在上方**（>80%）：多头强势控盘
- **主要在上方**（>60%）：多头占优
- **上下纠缠**（40-60%）：多空激烈博弈，方向待选
- **主要在下方**（>40%在下方）：空头占优
- **长时间在下方**（>80%）：空头强势压制

**实战操作要点**：
- 价格在均价线上方 → 多头占优
- 价格跌破均价线且无法收复 → 弱势信号（三部曲核心规则）
- 价格高出均价线>3% → 先卖部分锁利
- 黄线向上+白线在上方 → 最强做多信号
- 黄线向下+白线在下方 → 最弱做空信号

### 卖出三部曲信号（v5.0 分级标注）

| 持仓类型 | 卖出条件 | 信号级别 |
|---------|---------|---------|
| 前日涨停股 | 开盘30分钟不红盘(低于昨收) → 卖出 | ⚠️ 强信号 |
| 前日未涨停股 | 开盘1小时不红盘 → 卖出 | ⚡ 中信号 |
| 冲高回落 | 盘中涨幅超2%后回落破均价线 → 卖出 | ⚠️ 强信号 |
| 尾盘 | 不涨停 → 卖出 | ⚡ 中信号 |

### 综合评分（v5.0 四维度100分制）

脚本输出75分（量能25+承接25+预期差25），Claude补充趋势分25分。

| 维度 | 满分 | 评分因子 |
|------|------|---------|
| 量能 | 25 | 早盘占比(8) + 早盘额比(9) + 竞价量比(8) |
| 承接力 | 25 | 量价信号(15) + VWAP关系(8) + 全天额比(2) |
| 预期差 | 25 | gap_type/level 映射（above=18, meet=15, no_signal=12, below_weak=10, below_mod=6, below_strong=2） |
| 趋势 | 25 | 60日线上且5日线向上=25, 5日线上=15, 5日线附近=10, 5日线下=5 |

### 换手率评级（v5.0 对齐实战标准）

| 换手率 | 评级 | 说明 |
|--------|------|------|
| <1% | 极度低迷 | 无人问津 |
| 1-3% | 低迷 | 交投清淡 |
| 3-7% | 正常 | 常规交易 |
| 7-15% | 活跃 | 资金关注 |
| 15-25% | 高度活跃 | 热门标的 |
| >25% | 异常放量 | 筹码剧烈交换，需警惕 |

注：换手率需结合实际流通市值（扣除大股东持仓后）判断。热门题材股真实换手率 = 名义换手率 ÷ (1 - 大股东持仓比例)。

### 实时量比

| 量比 | 含义 |
|------|------|
| >2.0 | 明显放量，资金活跃 |
| 1.5-2.0 | 温和放量 |
| 0.8-1.5 | 正常 |
| <0.8 | 缩量，资金观望 |

### 涨停/跌停检测

基于昨收价正确计算：
- 普通股: 昨收 × 1.10（涨停）/ × 0.90（跌停）
- ST股: 昨收 × 1.05 / × 0.95
- 科创/创业板: 昨收 × 1.20 / × 0.80

## 持仓文件

持仓数据存储在: `{baseDir}/data/portfolio.json`

持仓格式支持 `effective_float_shares` 字段（用于换手率计算）。

## 限制

- 仅支持A股（沪深北交所）
- 交易时段外获取的是收盘数据（有标注）
- 分时数据最多500条（约2-3个交易日），竞价量数据依赖09:25数据是否在范围内
- 换手率需要流通股数数据，可通过 cn-financial 获取
- 趋势评分需要历史日K线，由 Claude 在报告生成时通过 cn-financial 计算
