## 数据源职责（严格遵守，禁止混用）

每个数据源有明确的职责边界，按步骤获取，不越界调用：

| 数据源 | 职责 | 获取内容 | 调用方式 |
|--------|------|---------|---------|
| **mx-data** | 竞价量 | 今日/前日集合竞价成交量、成交额 | `python mx_data.py "<股票名> 集合竞价数据"` |
| **cn-financial** | 行情+K线 | 实时行情、分时K线、历史日K线 | `mcp__cn-financial__get_*` 系列工具 |
| **mx-search** | 资讯解读 | 大盘情绪、板块热度、龙虎榜、公司信息（行业/主营/估值/财务）、研报新闻 | `python mx_search.py "<问句>"` |
| **analyze.py** | 量能分析 | 分时量能结构、VWAP、承接力、综合评分、操作结论 | `uv run scripts/analyze.py <代码> --minute` |
| **Claude** | 融合研判 | 趋势评分、综合评分汇总、情景推演、操作建议 | 基于以上所有数据生成 |

**禁止行为：**
- 禁止用 cn-financial 获取大盘情绪/板块热度/龙虎榜/公司信息（职责属于 mx-search）
- 禁止用 mx-search 获取实时行情/历史K线（职责属于 cn-financial）
- 禁止跳过 mx-data 直接用其他数据源的竞价量
- 禁止将 mx-search 的新闻文本直接传给 analyze.py（analyze.py 只接受结构化 JSON）

## ⚠️ 行为准则（必须严格遵守）

**这些规则优先级最高，违反任何一条都会导致用户得到错误信息：**

1. **执行流程（单股6步，按顺序严格执行）**：
   - **Step 1 [mx-data]：获取竞价量**
     `python C:/Users/kense/.claude/skills/mx-data/mx_data.py "<股票名> 集合竞价数据"`
     从返回 `rawTable["100000000047336"]` 提取今日（index 0）和前日（index 1）的竞价量（单位：股）
   - **Step 2 [cn-financial]：获取行情+K线**
     - `mcp__cn-financial__get_realtime_quote` → 实时行情（现价、涨跌、成交量等）
     - `mcp__cn-financial__get_historical_price`（start_date=20天前, adjust="qfq"）→ 历史日K线（趋势评分+近期走势）
   - **Step 3 [analyze.py]：运行量能分析**
     `uv run {baseDir}/scripts/analyze.py <代码> --minute --auction-today-shares <今日股数> --auction-prev-shares <前日股数>`
     输出：分时量能结构、VWAP、承接力、综合评分（75分）、操作结论
   - **Step 4 [mx-search]：获取资讯解读**（推荐但可选，提升报告质量）
     - `python mx_search.py "今日A股大盘行情走势 上证指数 深证成指"` → 大盘情绪
     - `python mx_search.py "今日A股热门板块涨幅排名"` → 板块热度
     - `python mx_search.py "<股票名>龙虎榜数据"` → 龙虎榜解读（仅涨停/异动股）
     - `python mx_search.py "<股票名>公司基本信息 行业板块"` → 公司信息（行业/主营/估值/财务）
   - **Step 5 [Claude]：趋势评分**
     基于 Step 2 的历史K线，计算趋势分（满分25分）
   - **Step 6 [Claude]：生成融合报告**
     整合所有数据，按七大章节结构输出完整分析
   - 多股时：Step 1 分别执行；Step 2-3 可批量；Step 4 龙虎榜需分别查询
2. **脚本输出是数据源，不是最终报告**。`analyze.py` 输出的章节是原始数据，必须基于这些数据生成融合报告。禁止原样展示、摘要、省略或改写脚本输出。始终输出完整分析，不存在"精简版"。
3. **同时段对比必须包含量比和额比**，两个维度缺一不可。
4. **数据真实性原则（最高优先级）**：脚本输出中标注"无前日数据"、"无竞价数据"、"数据不足"的字段，禁止编造或推测。`missing_data` 不为空时必须在报告中列出缺失项。宁可告知用户数据不足，也不能用猜测的数据支撑操作建议。
5. **盘后复盘模式**：用户明确要求复盘或数据状态为"已收盘"时，主动提示 `--review` 参数（自动包含 `--minute`）。复盘模式下必须生成次日操作计划。
6. **近期走势章节不绘制走势图**，只输出关键价位和走势定性文字描述。

**为什么这些规则很重要**：用户在实际交易中依赖这些数据做买卖决策。如果省略关键章节（比如"竞价量对比"或"次日操作计划"），用户可能错过关键的交易信号，造成实际资金损失。

### mx-data 竞价量解析说明

mx-data 返回的 JSON 中，`rawTable` 字段结构如下：

```json
{
  "rawTable": {
    "100000000047336": ["719500", "564200", ...],  // 开盘集合竞价成交量（股）
    "100000000047337": ["5072万元", ...],           // 开盘集合竞价成交额
    "100000000041102": ["75.27万股", ...],          // 收盘集合竞价成交量
    "headName": ["2026-05-18", "2026-05-15", ...]  // 日期（index 0 = 最新）
  }
}
```

提取规则：`rawTable["100000000047336"][0]` = 今日竞价量（股），`[1]` = 前日竞价量（股）。
直接将字符串转为整数传入 `--auction-today-shares` 和 `--auction-prev-shares`。

## 盘中看盘场景

盘中分析的核心决策链（按优先级）：

1. **竞价量比** → 判断今日开盘强弱基础（脚本"竞价量对比"章节）
2. **预期差信号** → 判断是否存在博弈机会（脚本"预期差信号"章节）
3. **早盘30分占比+额比** → 确认主力参与度（脚本"早盘量能预期"章节）
4. **VWAP关系** → 实时买卖时机（白线在黄线上方=多头占优）
5. **综合评分** → 最终持仓/离场决策（脚本"综合评分"章节 + Claude补充趋势分）

关键阈值详见下方"分时量能分析说明"。完整规则（买入三原则、三部曲例外、龙虎榜解读、仓位风控等）详见 `references/short-term-trading-tactics.md`——分析中遇到不确定的判断时应主动查阅。

## 盘后复盘场景

**触发条件**：用户明确要求复盘，或数据状态为"已收盘"。

**命令**：`uv run {baseDir}/scripts/analyze.py <代码> --review`

**复盘分析流程**：
1. 运行 `--review` 获取脚本复盘数据章节
2. 使用 `mcp__cn-financial__get_historical_price` 获取20日历史K线，计算趋势分
3. 汇总综合评分（100分制 = 脚本75分 + 趋势25分）
4. 读取 `references/short-term-trading-tactics.md`，结合其中的三部曲纪律、低吸原则、缩量决策树生成次日操作计划

**趋势分计算规则**（Claude 基于历史K线计算）：
- 价格在60日线上且5日线向上 → 25分
- 价格在5日线上但低于60日线 → 15分
- 价格在5日线附近（±1%）→ 10分
- 价格在5日线下 → 5分

**盘后标的筛选标准**（次日关注股）：
- 综合评分 ≥ 60/100
- 今日涨停或涨幅 > 5%
- 5日线向上
- 属于近期热点板块

## 分时量能分析说明

### 成交分布时段

| 时段 | 说明 |
|------|------|
| 早盘30分 (9:30-10:00) | 主力早盘动作，>30%为抢筹信号，<20%全天基本无戏 |
| 上午中段 (10:00-11:30) | 正常交易 |
| 下午中段 (13:00-14:30) | 正常交易 |
| 尾盘30分 (14:30-15:00) | >25%为尾盘异动信号 |

### 竞价量对比

集合竞价量来自新浪分时数据 09:25 那根K线。

| 竞价量比 | 信号 | 含义 |
|---------|------|------|
| >1.3 | 放量开盘（强） | 承接力强，冲高概率高 |
| 1.0-1.3 | 温和放量（中） | 有一定承接 |
| 0.7-1.0 | 平量开盘（中） | 强弱待观察 |
| <0.7 | 缩量开盘（弱） | 不好做T，冲高概率低 |

### 预期差检测

仅在前日涨停时有实质意义。

| 条件 | 预期差类型 | 操作参考 |
|------|-----------|---------|
| 前日涨停+今日低开>2% | below/strong | 30分钟不翻红立即卖出 |
| 前日涨停+今日低开+竞价缩量 | below/moderate | 密切关注30分钟内能否翻红 |
| 前日涨停+今日小幅低开 | below/weak | 等翻红后量能确认再介入 |
| 前日涨停+今日小幅高开 | meet | 正常观察 |
| 前日涨停+今日高开>3% | above | 高开不追，等回踩 |

### 同时段对比（量比+额比双维度）

| 额比范围 | 趋势判断 | 含义 |
|----------|----------|------|
| >1.3 | 明显增量 | 资金积极参与 |
| 1.1-1.3 | 温和增量 | 量能温和放大 |
| 0.9-1.1 | 基本持平 | 量能稳定 |
| 0.7-0.9 | 温和缩量 | 资金观望 |
| <0.7 | 明显缩量 | 资金离场或惜售 |

额比比量比更重要：价格上涨时成交额同步放大，额比能更真实反映资金参与度。

### 承接力判断（多维验证）

基于昨收价（而非当日第一根K线）判断量价配合：

| 信号 | 条件 | 判断 |
|------|------|------|
| 缩量回调 | 跌>0.5% + 额比<0.8 | 承接力强（主力护盘，散户惜售） |
| 放量下跌 | 跌>0.5% + 额比>1.2 | 承接力弱（主力出货，恐慌抛售） |
| 放量上涨 | 涨>0.5% + 额比>1.2 | 量价配合好（主力积极做多） |
| 缩量上涨 | 涨>0.5% + 额比<0.8 | 量价背离（主力控盘或散户推动） |

### VWAP（均价线）增强分析

成交量加权均价（分时图黄线），是实战中最重要的参考线。

**黄线走势方向判断**：
- **黄线向上**：均价成本持续上移，买方主动，积极做多
- **黄线横盘**：均价成本稳定，多空平衡，等待方向选择
- **黄线向下**：均价成本下移，卖方主动，谨慎观望
- **黄线震荡**：均价成本波动较大，多空激烈博弈

**白黄线关系判断**：
- **长时间在上方**（>80%）：多头强势控盘
- **主要在上方**（>60%）：多头占优
- **上下纠缠**（40-60%）：多空激烈博弈，方向待选
- **主要在下方**（>40%在下方）：空头占优
- **长时间在下方**（>80%）：空头强势压制

**实战操作要点**：
- 价格在均价线上方 → 多头占优
- 价格跌破均价线且无法收复 → 弱势信号（三部曲核心规则）
- 价格高出均价线>3% → 先卖部分锁利
- 黄线向上+白线在上方 → 最强做多信号
- 黄线向下+白线在下方 → 最弱做空信号

### 卖出三部曲信号（分级标注）

| 持仓类型 | 卖出条件 | 信号级别 |
|---------|---------|---------|
| 前日涨停股 | 开盘30分钟不红盘(低于昨收) → 卖出 | ⚠️ 强信号 |
| 前日未涨停股 | 开盘1小时不红盘 → 卖出 | ⚡ 中信号 |
| 冲高回落 | 盘中涨幅超2%后回落破均价线 → 卖出 | ⚠️ 强信号 |
| 尾盘 | 不涨停 → 卖出 | ⚡ 中信号 |

### 综合评分（四维度100分制）

脚本输出75分（量能25+承接25+预期差25），Claude补充趋势分25分。

| 维度 | 满分 | 评分因子 |
|------|------|---------|
| 量能 | 25 | 早盘占比(8) + 早盘额比(9) + 竞价量比(8) |
| 承接力 | 25 | 量价信号(15) + VWAP关系(8) + 全天额比(2) |
| 预期差 | 25 | gap_type/level 映射（above=18, meet=15, no_signal=12, below_weak=10, below_mod=6, below_strong=2） |
| 趋势 | 25 | 60日线上且5日线向上=25, 5日线上=15, 5日线附近=10, 5日线下=5 |

### 换手率评级

| 换手率 | 评级 | 说明 |
|--------|------|------|
| <1% | 极度低迷 | 无人问津 |
| 1-3% | 低迷 | 交投清淡 |
| 3-7% | 正常 | 常规交易 |
| 7-15% | 活跃 | 资金关注 |
| 15-25% | 高度活跃 | 热门标的 |
| >25% | 异常放量 | 筹码剧烈交换，需警惕 |

注：换手率需结合实际流通市值（扣除大股东持仓后）判断。热门题材股真实换手率 = 名义换手率 ÷ (1 - 大股东持仓比例)。

### 实时量比

| 量比 | 含义 |
|------|------|
| >2.0 | 明显放量，资金活跃 |
| 1.5-2.0 | 温和放量 |
| 0.8-1.5 | 正常 |
| <0.8 | 缩量，资金观望 |

### 涨停/跌停检测

基于昨收价正确计算：
- 普通股: 昨收 × 1.10（涨停）/ × 0.90（跌停）
- ST股: 昨收 × 1.05 / × 0.95
- 科创/创业板: 昨收 × 1.20 / × 0.80

### 大盘共振判断（新增）

预期差分析的前提条件。大盘背景影响个股信号的可靠性：

| 大盘状态 | 对个股的影响 | 操作建议 |
|---------|------------|---------|
| 强势普涨(>1.5%) | 预期差信号更可靠，低位股反弹概率高 | 积极参与 |
| 温和上涨(0.5-1.5%) | 操作环境正常 | 正常操作 |
| 震荡(-0.5~0.5%) | 个股分化，需精选 | 谨慎操作 |
| 温和下跌(-1.5~-0.5%) | 市场偏冷，逆势股风险高 | 减少操作 |
| 恐慌下跌(<-1.5%) | 系统性风险，泥沙俱下 | 空仓观望 |

数据来源：`mcp__cn-financial__get_market_overview`（上证指数+创业板指涨跌幅）

### 板块热度判断（新增）

"低吸三原则"第一条：必须是近期热点。板块热度影响个股资金关注度：

| 板块热度 | 板块涨幅 | 对个股的影响 |
|---------|---------|------------|
| 极热 | >2% | 资金集中涌入，龙头股优先 |
| 偏热 | 1-2% | 资金关注度高，可参与 |
| 正常 | 0-1% | 正常交易 |
| 偏冷 | -1-0% | 资金流出，谨慎 |
| 冷 | <-1% | 板块弱势，不介入 |

数据来源：`mcp__cn-financial__get_industry_list`（行业板块涨跌幅排名）

### 涨停深度分析（新增）

涨停股的细分分析，提供更精确的操作指导：

| 分析维度 | 指标 | 判断标准 |
|---------|------|---------|
| 封板时间 | 首次触及涨停价的时间 | 早盘(≤10:00)=强，盘中=中，尾盘=弱 |
| 封板稳定性 | 涨停价附近波动次数 | 多次开板=不稳定，警惕出货 |
| 连板判断 | 前日是否也涨停 | 2连板+，需关注情绪变化 |
| 封单强度 | 尾盘成交量占比 | 尾盘放量缩量可辅助判断 |

### 龙虎榜解读（新增）

游资核心参考数据。解读要点：

| 特征 | 次日预判 |
|------|---------|
| 敢死队占2席+买入超总额1/3 | 次日高开砸盘，高开不追 |
| 买1独大（是买2-5的几倍） | 吃独食，低开可进，高开当心 |
| 买1-5金额平均，无敢死队 | 相对安全，有肉吃 |
| 卖出比买入大50%以上 | 小心次日继续下跌 |

席位关键词：金田(强势)、淮海(转折)、佛山(堵单)、成都/校长(弱势)、山东帮(妖股行情)

数据来源：`mcp__cn-financial__get_dragon_tiger`

### 承接力量化指标（新增）

基于实战手册的量化标准，更精确判断承接力：

| 指标 | 计算方式 | 阈值 | 含义 |
|------|---------|------|------|
| 半小时量/前一天全天 | open_30_vol / prev_day_total | ≥100% | 强承接，大概率拉升 |
| 1小时量/前一天全天 | open_60_vol / prev_day_total | ≥60% | 承接力良好 |
| 同期量/前日同期 | today_same / prev_same | <50% | 缩量，承接不足 |

### 三部曲特殊情况（新增）

实战手册中的例外规则，避免机械执行：

| 情况 | 处理 |
|------|------|
| 开盘-4%但分时持续向上+黄线向上 | 可适当等待，不急于止损 |
| 成交额>50亿的趋势大票 | 持有到尾盘，破5日线才走 |
| 庄股/妖股 | 坚持到尾盘，但涨40-50%后不适用 |
| 大利空/快速跌停(-9%) | 立即止损，不等 |

## 限制

- 仅支持A股（沪深北交所），使用6位数字代码，系统自动识别市场：沪市(6开头)、深市(0/3开头)、北交所(8/4开头)、科创板(68开头，±20%)、创业板(30开头，±20%)
- 交易时段外获取的是收盘数据（有标注）
- 分时数据最多500条（约2-3个交易日），竞价量数据依赖09:25数据是否在范围内
- 换手率需要流通股数数据，可通过 cn-financial 获取
- 趋势评分需要历史日K线，由 Claude 在报告生成时通过 cn-financial 计算
